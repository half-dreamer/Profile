# DEV_SPEC - 个人主页项目开发规格说明

## 1. 项目简介

本项目用于构建一个面向求职与个人品牌展示的个人网站，支持发布到 GitHub Pages。  
网站采用三页面结构：

- `关于我`：展示个人简介、学历与工作经历时间线
- `学习文章`：展示知乎技术文章入口、文章列表、技术栈
- `生活记录`：展示旅游记录、日记、心得

目标是实现“网页端优先、风格简洁酷炫、内容可持续维护”的个人主页。

## 2. 技术架构

### 2.1 架构类型
- 前后端分离中的“纯前端静态架构”（无自建后端）
- 三页面 + 共享组件的混合 MPA 架构（页面独立，逻辑复用）
- 构建产物为静态资源，直接托管到 GitHub Pages
- 数据驱动渲染：页面逻辑层与内容数据层完全解耦

### 2.2 目录策略（文档统一放根目录）
- `README.md`：项目说明和使用指南
- `DEV_SPEC.md`：开发规格文档（本文件）
- `TEST_PLAN.md`：测试文档
- 页面与资源文件建议按职责拆分（文档仍放根目录）：
  - `about.html`、`learning.html`、`life.html`：页面入口
  - `src/js/`：渲染逻辑、交互逻辑、工具函数
  - `src/styles/`：主题变量与页面样式
  - `src/config/`：数据配置文件（个人信息、经历、文章、生活记录）
  - `public/assets/`：图片、图标、简历文件

### 2.3 分层设计（逻辑与数据解耦）

1. 展示层（View）
   - 负责页面结构与视觉容器，不直接写死业务数据
   - 文件：`about.html`、`learning.html`、`life.html`
2. 逻辑层（Renderer + Interaction）
   - 负责把配置数据渲染为 UI，并处理导航高亮、筛选、展开等交互
   - 文件：`src/js/renderers/*.ts`、`src/js/pages/*.ts`
3. 数据层（Config）
   - 负责维护内容，不包含 DOM 操作
   - 文件：`src/config/site.config.ts`、`timeline.config.ts`、`articles.config.ts`、`life.config.ts`
4. 校验层（Validation，可选但推荐）
   - 启动时校验配置结构，避免字段缺失导致页面渲染异常
   - 文件：`src/js/validators/configValidator.ts`

### 2.4 先进技术栈

- **构建工具**：Vite（快速开发与构建）
- **语言**：TypeScript（提升可维护性与类型安全）
- **样式体系**：Tailwind CSS + CSS Variables（高效开发 + 主题统一）
- **代码质量**：ESLint + Prettier（规范与格式统一）
- **测试**：Vitest（单元测试）+ Playwright（关键流程 E2E）
- **配置校验**：Zod（配置文件 schema 校验，确保数据合法）
- **部署**：GitHub Actions + GitHub Pages（自动化发布）

### 2.5 数据解耦与配置化策略（强制要求）

1. 所有业务内容必须放在配置文件，不允许散落在页面模板中。  
2. 页面渲染函数只接收配置对象，不直接依赖硬编码文案。  
3. 配置文件拆分建议：
   - `src/config/site.config.ts`：站点基础信息（姓名、定位、链接）
   - `src/config/timeline.config.ts`：学历与工作经历
   - `src/config/articles.config.ts`：学习文章与知乎链接
   - `src/config/life.config.ts`：旅游/日记/心得
4. 配置变更无需改动渲染逻辑：只改配置即可更新页面内容。  
5. 发布前执行配置校验：字段缺失时给出明确报错或兜底默认值。

## 6. 代码规范

以前端设计与实现涉及的代码规范为准，统一执行以下规则：

1. 语义化与结构规范
   - 使用语义化标签（如 `header`、`nav`、`main`、`section`、`footer`）
   - 页面结构清晰，模块边界明确，避免无意义嵌套
2. 命名规范
   - 文件名使用小写与短横线风格（如 `learning.html`）
   - CSS 类名采用统一前缀或 BEM 风格，避免命名冲突
   - JavaScript 函数采用动词开头的驼峰命名（如 `renderTimeline`）
3. 样式规范
   - 颜色、间距、阴影、圆角等设计 token 使用 CSS 变量集中管理
   - 统一排版系统（标题层级、正文行高、字重）
   - 动效遵循“轻、快、少”，避免影响可读性
4. JavaScript 规范
   - 以小函数拆分逻辑，一个函数只做一件事
   - 渲染逻辑与数据定义分离，避免在 HTML 中硬编码大量内容
   - 对缺失数据提供兜底文案，避免页面报错
5. 响应式规范
   - 以网页端（PC）为优先，再兼容移动端
   - 断点行为可预测，避免布局跳变
6. 文档与可维护性规范
   - 每次新增模块需同步更新 `README.md`、`DEV_SPEC.md`、`TEST_PLAN.md`
   - 关键函数添加简短注释，说明输入、输出与用途
