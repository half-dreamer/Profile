# 测试文档 - TEST_PLAN

## 1. 测试目标

验证个人主页项目在网页端优先场景下的可用性、正确性和可发布性，确保可稳定部署到 GitHub Pages。

## 2. 测试范围

- 页面：`about.html`、`learning.html`、`life.html`
- 公共资源：`styles.css`、`script.js`、内容数据文件
- 外部链接：知乎主页、文章链接、简历下载链接

## 3. 测试环境

- macOS + Chrome（主）
- macOS + Safari（辅）
- GitHub Pages 线上地址（发布后）

## 4. 测试用例

### 4.1 页面与导航
- 打开每个页面，页面不报错、不空白
- 顶部导航可以互相跳转
- 当前页面导航项高亮正确

### 4.2 关于我页面
- 个人信息展示完整
- 学历时间线顺序正确（时间从近到远或远到近一致）
- 工作经历时间线顺序正确
- 时间线节点样式和交互（hover/展开）正常

### 4.3 学习文章页面
- 知乎主页按钮可跳转
- 3 篇文章卡片展示正确（标题、摘要、日期、标签）
- 文章卡片链接跳转到对应知乎文章
- 技术栈分组显示正确

### 4.4 生活记录页面
- 三类内容可见：旅游记录 / 日记 / 心得
- 每类卡片内容显示正常（标题、日期、摘要）
- 图片缺失时显示占位图，不影响布局

### 4.5 样式与交互
- PC 端排版整齐、风格统一、文本可读
- 动效流畅，无明显卡顿或闪烁
- 窗口缩放时布局不出现明显错位

### 4.6 部署与回归
- GitHub Pages 发布后可通过公网访问
- 首屏、导航、外链、核心内容在线上可用
- 刷新页面不出现 404（静态多页面路径正确）

## 5. 缺陷等级定义

- P0：页面无法打开、导航失效、核心内容不显示
- P1：核心模块功能异常、关键链接失效
- P2：样式或交互问题，不影响主流程
- P3：文案、间距、轻微视觉问题

## 6. 通过标准

- P0、P1 缺陷全部修复
- 核心流程（访问页面 -> 浏览内容 -> 点击链接）全部通过
- GitHub Pages 线上地址可稳定访问
